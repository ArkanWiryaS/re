# BAB II: ANALISIS PENGGUNA DAN TUGAS

## 2.1 Metodologi Analisis Pengguna

### 2.1.1 Pendekatan User Research

Analisis pengguna dalam penelitian ini menggunakan pendekatan mixed-methods yang menggabungkan qualitative dan quantitative research methods untuk memperoleh pemahaman mendalam tentang target users. Metodologi yang diterapkan mencakup user interviews dengan representative users dari kedua user groups, observational studies untuk memahami current workflows dan pain points, serta survey analysis untuk mengidentifikasi patterns dan trends dalam user behavior dan preferences.

Proses user research dilaksanakan melalui structured interviews dengan 15 potential users, terdiri dari 8 innovators dan 7 investors, untuk mengidentifikasi core needs, motivations, dan barriers yang mereka hadapi dalam current ecosystem. Observational studies dilakukan untuk memahami bagaimana target users currently discover opportunities, evaluate projects, dan make decisions dalam konteks innovation-investment collaboration.

### 2.1.2 Framework Analisis Multi-Tier

Platform Verdant dirancang untuk melayani dual user groups dengan charakteristik dan kebutuhan yang berbeda, sehingga diperlukan framework analisis multi-tier yang dapat mengakomodasi kompleksitas ini. Framework yang dikembangkan mencakup primary user classification berdasarkan roles dan goals, secondary classification berdasarkan technical proficiency dan domain expertise, serta tertiary classification berdasarkan experience level dan engagement patterns.

Analisis multi-tier ini memungkinkan identification of shared needs dan unique requirements untuk setiap user segment, facilitating design decisions yang dapat provide value untuk multiple user groups tanpa compromising user experience. Framework ini juga mempertimbangkan intersection points dimana kedua user groups berinteraksi dalam platform, memastikan seamless collaboration experience.

### 2.1.3 Data Collection Methods

Data collection dilakukan melalui multiple touchpoints untuk memastikan comprehensive understanding tentang target users. Semi-structured interviews dengan duration 45-60 menit dilakukan untuk menggali insights mendalam tentang current processes, pain points, dan expectations. User journey mapping sessions dilakukan untuk memahami end-to-end experience dalam innovation-investment processes.

Survey deployment kepada wider population (50 innovators dan 40 investors) memberikan quantitative validation terhadap findings dari qualitative research. Competitive analysis terhadap existing platforms seperti AngelList, Gust, dan SeedInvest dilakukan untuk memahami current market solutions dan identify gaps yang dapat diaddress oleh platform Verdant.

### 2.1.4 User Segmentation Strategy

User segmentation strategy dikembangkan berdasarkan behavioral patterns, goals, dan technical capabilities untuk memastikan platform design dapat accommodate different user needs effectively. Primary segmentation dilakukan berdasarkan user roles (Innovators vs Investors), secondary segmentation berdasarkan experience level dan domain expertise, serta tertiary segmentation berdasarkan engagement frequency dan platform usage patterns.

Segmentation strategy ini mempertimbangkan user lifecycle dalam platform, mulai dari onboarding hingga advanced usage scenarios, memastikan bahwa design dapat support user growth dan evolving needs over time. Cross-segment analysis juga dilakukan untuk identify collaboration opportunities dan interaction patterns antara different user segments.

## 2.4 Hierarchical Task Analysis

### 2.4.1 Framework HTA untuk Dual User Groups

Hierarchical Task Analysis (HTA) untuk platform Verdant dikembangkan dengan considering complex interaction patterns antara innovators dan investors dalam collaborative ecosystem. Framework HTA menggunakan goal-oriented approach yang decompose high-level objectives menjadi specific tasks dan subtasks yang dapat be addressed melalui platform functionality.

Primary goals untuk innovators include project showcasing, investor discovery, partnership building, dan funding acquisition. Primary goals untuk investors include project discovery, due diligence execution, network building, dan investment management. Cross-cutting goals include communication facilitation, knowledge sharing, dan ecosystem development yang require collaborative functionality.

HTA framework considers both individual user tasks dan collaborative tasks yang involve interaction antara multiple users. Task hierarchy organizes dari high-level strategic goals ke operational tasks dan specific interface interactions, memungkinkan systematic design approach yang ensures all user needs dapat be addressed effectively.

### 2.4.2 Goal Level 0: Platform Engagement

Pada highest level, platform engagement goal mencakup establishing presence dalam sustainable technology ecosystem melalui profile creation dan content development. Untuk innovators, ini meliputi creating compelling project presentations yang can attract investor interest, building credibility melalui comprehensive documentation, dan establishing thought leadership dalam specific technology domains.

Untuk investors, goal level 0 mencakup building reputation sebagai active sustainable technology investor, accessing high-quality deal flow, dan contributing expertise untuk ecosystem development. Shared goals include network expansion, knowledge acquisition, dan ecosystem contribution yang require both individual activities dan collaborative engagement.

Success metrics untuk goal level 0 include profile completeness scores, engagement metrics, network growth indicators, dan platform contribution measurements yang can be tracked dan optimized over time untuk improve overall platform value proposition.

### 2.4.3 Goal Level 1: Core Platform Functions

Core platform functions untuk innovators include comprehensive project management capabilities yang allow untuk detailed project documentation, milestone tracking, dan progress reporting. Investor discovery functionality enables targeted search dan outreach capabilities dengan sophisticated filtering options berdasarkan investment criteria, geographic preferences, dan sector focus areas.

Technology demonstration capabilities merupakan unique value proposition yang allows innovators untuk showcase AI dan technology solutions melalui interactive demos, video presentations, dan detailed technical documentation. Communication tools facilitate direct interaction dengan investors melalui messaging systems, meeting scheduling, dan document sharing capabilities.

Untuk investors, core functions include advanced project discovery dengan filtering dan recommendation systems, comprehensive due diligence tools untuk technical dan business evaluation, portfolio management capabilities untuk tracking investments dan monitoring progress, dan market intelligence tools untuk staying updated dengan industry trends dan competitive landscape.

### 2.4.4 Goal Level 2: Specific Task Execution

Specific task execution mencakup detailed workflows untuk completing core platform activities efficiently dan effectively. Untuk project submission process, ini meliputi step-by-step guidance untuk creating comprehensive project profiles, uploading supporting documents, setting up interactive demos, dan optimizing content untuk search dan discovery.

Investor outreach workflows include target investor identification menggunakan platform search dan filtering capabilities, personalized message composition dengan template support, follow-up tracking, dan relationship management tools. Due diligence support includes structured evaluation frameworks, collaborative review processes, dan decision tracking capabilities.

AI demonstration setup requires specific technical workflows untuk integrating TensorFlow.js demos, configuring interactive features, providing user guidance, dan ensuring optimal performance across different devices dan browsers. Communication workflows include meeting scheduling, document sharing, feedback collection, dan project update distribution yang facilitate ongoing collaboration.

## 2.6 User Journey Mapping

### 2.6.1 Innovator Journey: From Idea to Investment

Innovator user journey dimulai dengan awareness stage dimana potential users discover platform Verdant melalui various touchpoints including industry events, social media, referrals, atau search engines. Initial landing experience focuses pada communicating unique value proposition dengan clear differentiation dari existing platforms, emphasizing AI demonstration capabilities dan sustainable technology focus.

Onboarding process guides users melalui account creation, profile setup, dan initial project creation dengan step-by-step assistance dan best practice recommendations. Progressive profiling approach allows users untuk gradually build comprehensive profiles tanpa overwhelming initial experience, dengan ongoing prompts untuk enhance profile completeness sebagai user engagement increases.

Project development phase includes iterative content creation dengan platform guidance untuk optimizing project presentations, setting up AI demonstrations, creating compelling narratives, dan building evidence base untuk investor attraction. Platform provides feedback mechanisms dan optimization suggestions untuk improving project visibility dan appeal.

Investor engagement phase facilitates discovery oleh relevant investors melalui platform algorithms, direct outreach capabilities, dan networking event integrations.
